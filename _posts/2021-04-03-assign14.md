# 오픈 API 로 기능 1가지 기획하기

이번에 만들고자 하는 서비스는, 가칭 "이사 어디가?" 이다.

풀고자 하는 문제는, 나처럼 지방 혹은 다른 지역에서 서울로 이사를 와야 하는 상황에 있는 사람들이며, 

지리에 대한 사전지식이 별로 없는 (가령 서울에 예전에 부모님이 근무를 해서 어느 동네가 좋다 라는걸 모르는 상황) 경우를 가정했다.

<img src='https://user-images.githubusercontent.com/6457691/113472308-5a424180-949d-11eb-927c-372d96c57bd2.png' width = '500'>

사용하는 api는 크게 4가지이다. 

첫번째는 버스의 정보, 두번째는 집의 가격 정보, 세번째는 주소 정보, 마지막으로 지하철 정보 api이다.

있는지 몰라서 작성은 안했는데 만약 부동산 매물 정보도 있다면... (네이버 부동산 같은) 사용하면 좋을 것 같다.

<img src='https://user-images.githubusercontent.com/6457691/113472314-63331300-949d-11eb-93b0-0e5a69137cd2.png' width = '500'>

만들고자 하는 서비스의 예상 방향이다.


간단하게, 지도의 특정 위치를 찍으면 혹은 주소를 입력하면 해당 지점 근처의 버스 정류소를 조회하여 

노선이 지나가는 지점 별로 거리, 지가, 출퇴근 시간, 면적 과 같은 정보들을 취합하여 비교 할 수 있게 보여준다.

<img src='https://user-images.githubusercontent.com/6457691/113472319-662e0380-949d-11eb-85ee-f568eb5a3f8f.png' width = '500'>

마지막으로 비즈니스 모델을 간단하게 그려봤다.

<img src='https://user-images.githubusercontent.com/6457691/113472321-67f7c700-949d-11eb-98f0-eb25b4e0df3d.png' width = '500'>

역시 요즘의 트렌드 ESG, 혹은 공공의 발전, 데이터 사업등. 아무튼 이런게 최고다.
